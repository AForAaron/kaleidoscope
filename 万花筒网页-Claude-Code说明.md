# 项目说明：网页版万花筒（支持倾斜陀螺仪玩法）

## 一句话目标
帮我做一个**单文件网页版万花筒**，主要给小朋友（小学生）在 iPad 上玩。核心玩法是手指在屏幕上拖动产生彩色花纹，并且**倾斜 iPad 时花纹会跟着旋转**（用陀螺仪）。

## 技术要求
- **一个独立的 `index.html` 文件**搞定全部，内联 CSS 和 JS，不要用任何构建工具、不要 npm、不要外部依赖（不引入任何第三方库）。
- 用 **Canvas 2D** 实现即可（性能足够，不用 WebGL，保持简单）。
- 代码加好中文注释，方便我以后自己改。
- 做完后告诉我怎么在本地直接双击打开测试，以及怎么部署到一个免费静态托管（比如 GitHub Pages / Vercel / Netlify 任选一个，给最简单的步骤）。

## 万花筒核心效果（重点，请认真实现）
1. 把整个圆分成 N 个对称扇区（默认 **12** 个，即每 30°一个），形成镜像对称的经典万花筒效果。
2. 做法建议：先在一个扇区（楔形区域）里绘制"源图案"，然后通过旋转 + 镜像翻转，把这一个扇区复制平铺成完整的圆，这样相邻扇区互为镜像，看起来才像真正的万花筒。
3. "源图案"来自两部分叠加：
   - **用户手指拖动**留下的彩色轨迹（粗一点的圆头线条，颜色鲜艳）。
   - 一些**自动缓慢飘动的彩色形状/圆点**作为背景，让画面即使不碰也一直在动。
4. 整体画面应该**持续缓慢自动旋转**，加上倾斜带来的旋转，两者叠加。

## 倾斜陀螺仪玩法（最容易做错，务必按这里来）
- 用浏览器的 `DeviceOrientationEvent` 监听设备方向，用 `beta`（前后倾）和 `gamma`（左右倾）来控制万花筒整体的旋转角度。
- **iOS 13+ 的关键坑**：在 iPhone/iPad 上，`DeviceOrientationEvent` 必须由用户的一次**点击/触摸手势**触发 `DeviceOrientationEvent.requestPermission()` 来请求授权，否则陀螺仪完全没反应。
  - 所以请在进入页面时显示一个大大的**「开始」按钮**，用户点击它时：① 调用 `requestPermission()` 申请陀螺仪权限；② 如果不支持该方法（安卓/旧设备）就直接开始监听；③ 然后再进入万花筒主界面。
  - 如果用户拒绝授权或设备没有陀螺仪，要能**优雅降级**：万花筒照常自动旋转 + 手指拖动玩，不要报错、不要白屏。
- 倾斜带来的旋转要做**平滑插值（lerp）**，不要让画面随手抖动而剧烈晃，体验要顺滑。

## 交互设计（面向小朋友，越简单越好）
- 全屏显示，没有复杂菜单和文字说明。
- **手指拖动** = 画彩色花纹。
- **单指点击（非拖动）** = 切换一套新的配色方案（准备 4~6 套鲜艳的儿童友好色板，循环切换）。
- 一个不显眼的小角落放一个**清空按钮**（图标即可，比如圆形箭头），点了把画布擦干净重新开始。
- 触摸目标要大，适合小朋友的手指。

## 视觉与体验
- 配色鲜艳、活泼、对比强，适合小孩。
- 背景纯黑或深色，让彩色花纹更突出。
- 画布要**自适应屏幕大小**（监听 resize），并处理高分屏（用 `devicePixelRatio` 保证清晰不糊）。
- 做成"可添加到主屏幕"的网页 app 体验：在 `<head>` 里加上
  - `<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no, viewport-fit=cover">`
  - `<meta name="apple-mobile-web-app-capable" content="yes">`
  - `<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">`
  - 一个简单的 emoji 或内联 SVG 作为图标（可选）。
- 禁止页面在触摸拖动时滚动 / 橡皮筋效果（`touch-action: none`，并在触摸事件里 `preventDefault`）。

## 性能
- 用 `requestAnimationFrame` 做主循环。
- 注意手指拖动产生的轨迹点不要无限增长导致卡顿（比如限制保留的点数量，或定期把轨迹"烘焙"进一张离屏 canvas）。
- 目标是在 iPad 上流畅运行（60fps 最好，不卡顿为底线）。
